# こもれび商店 — 販売×管理が連動するECサイト（サンプル）

**ストア（販売）** と **管理画面（ダッシュボード）** が、**1つのデータベースで連動**するECサイトのサンプルです。
ストアで注文すると在庫が減り、その注文が管理画面の売上グラフ・注文一覧・在庫に**即反映**されます。

- 🛒 ストア（販売）: https://nkikuchisg-commits.github.io/commerce-suite/
- 📊 管理画面（ダッシュボード）: https://nkikuchisg-commits.github.io/commerce-suite/admin.html
  - デモ用パスワード：`demo`（ログイン画面に表示・そのままログイン可）

## できること

### ストア側（販売）
- 商品一覧（カテゴリ絞り込み・並べ替え）・**在庫表示（残りわずか／在庫切れ）**
- カート（数量変更・送料計算・5,000円以上で送料無料）・注文手続き・注文完了
- 注文するとデータベースに記録され、**在庫が減る**

### 管理側（ダッシュボード）
- **ダッシュボード**：売上高・注文数・平均注文額・未発送件数（前期比つき）＋
  売上推移（折れ線）・カテゴリ構成比（ドーナツ）・人気商品TOP5（棒）・最近の注文。期間切替（7/30/90日）
- **注文管理**：注文一覧を状態でしぼり込み、プルダウンで状態変更（未発送→発送済み→完了）
- **商品・在庫**：在庫の増減・在庫アラート・累計販売数・公開/非公開の切替
- 「デモデータを初期化」ボタンで初期状態に戻せます

→ 売上の数字や在庫はすべて、**ストアの注文と同じデータ**から計算しています。

## このサイトは「静的」？「動的」？

- **ホスティングは静的**です（GitHub Pagesがファイルを返すだけ。WordPress等のサーバー処理はありません）。
- **動作は動的**です（カート・在庫・集計は**ブラウザの中のJavaScript**が行います）。
- つまり「静的ホスティング上で、JavaScriptが動的に動く」構成。**WordPressではありません。**

## データベースについて（MySQL用ファイル同梱）

動くデモは、サーバー不要で誰でも試せるよう**ブラウザ内のストレージ（localStorage）**をデータベース代わりにしています（`db.js`）。
一方で、**同じデータ構造をMySQLで設計したファイル**を同梱しています。実務／学習用の本物のSQLです。

| ファイル | 内容 |
|---|---|
| `schema.sql` | テーブル設計（products / customers / orders / order_items、主キー・外部キー・索引つき） |
| `seed.sql` | サンプルデータ（商品12点＋注文の例） |
| `queries.sql` | ダッシュボードの各数字を出すSQL（売上推移・カテゴリ別・人気商品・在庫アラート 等）＋在庫を減らすトランザクション例 |

```bash
# MySQLでの実行例
mysql -u root -p -e "CREATE DATABASE komorebi CHARACTER SET utf8mb4;"
mysql -u root -p komorebi < schema.sql
mysql -u root -p komorebi < seed.sql
mysql -u root -p komorebi < queries.sql   # 集計結果を確認
```

> 本番のWebサイトでMySQLを使う場合は、MySQLが動くサーバー（＋PHPやNode等のプログラム）が必要です。
> 静的ホスティング（GitHub Pages）だけではMySQLは動かせないため、デモはブラウザ内DBで実装しています。

## ファイル構成

```
commerce-suite/
├── index.html / store.js   … ストア（販売）
├── admin.html / admin.js   … 管理画面（ダッシュボード・注文管理・在庫管理）
├── db.js                   … 共有データ層（商品・注文・在庫／localStorage・初期データ生成）
├── style.css               … 共通スタイル
├── schema.sql / seed.sql / queries.sql … MySQL用の設計・データ・集計クエリ
```

## 技術

- HTML / CSS / バニラJavaScript（フレームワーク・グラフライブラリ不使用）
- 共有データ層 `db.js`：商品（在庫）・注文を localStorage に保存。初回は擬似乱数で過去120日分の注文を生成
- ストアの注文 → `DB.placeOrder()` で注文記録＋在庫減 → 管理画面が同じデータを集計
- グラフ（折れ線・棒・ドーナツ）はSVGを自前で描画
- 別タブで開いていても `storage` イベントで管理画面が自動更新
- サーバー不要・静的ホスティング（GitHub Pages）

## 受託での活用

「ECサイト＋管理画面」「受発注管理」「販売ダッシュボード」案件のサンプルです。
実運用では、データベース（MySQL/PostgreSQL等）・決済（Stripe等）・認証をサーバー側に置く構成になります。
本サンプルはその**画面・データ設計・集計ロジック**のデモで、MySQL設計（`schema.sql`／`queries.sql`）も提示しています。

## 注意

商品・価格・在庫・注文・顧客はすべて架空のサンプルです。決済・発送・実在庫の連動はありません。
